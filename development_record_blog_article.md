# AIと二人三脚！ブラウザで動く「ガラポン抽選アプリ」開発・改修記録

## はじめに
こんにちは！この記事では、ブラウザ上で動作する「ガラポン抽選アプリ」を、AIアシスタントと一緒に改修した際の実績と、技術的なアプローチ・直面した課題について振り返る開発記録をまとめています。

機能追加からレイアウトの崩れ修正、さらには複数PC間でのGit連携まで、盛り沢山の内容となりました。

---

## 1. 今回の要件（やりたかったこと）
元々あったガラポンアプリをベースに、以下の仕様変更と演出の強化を行いました。

1. **確率と結果のシンプル化**
   - 当たり（赤色）：20%（1万円割引クーポン）
   - ハズレ（白色）：80%（残念賞）
   の2種類に絞る。
2. **結果画像の表示**
   - 当たり時は `win.jpg`、ハズレ時は `lose.jpg` を結果ポップアップで画像として表示する。
3. **豪華な演出の追加**
   - 当たりが出た時だけ、画面いっぱいの「紙吹雪」エフェクトと、「和太鼓＋鉦（カネ）」のお囃子サウンドを鳴らして特別感を出す。

---

## 2. 開発のハードルと解決したアプローチ

### 最大の壁：画像が潰れて「細い線」になってしまう問題
開発を進める中で一番苦戦したのが、**結果の画像（`win.jpg`等）が表示されず、細い線のように縦に極端に潰れてしまう**というCSS/レイアウトの不具合でした。

**【原因の調査】**
画像を表示する親コンテナ自体のサイズ制約や、`hidden` クラス（Tailwind CSS）の切替タイミングが複雑に絡み合っていました。
特に画面サイズが小さいモバイル端末では、親要素の `overflow-hidden` やFlexboxの縮小設定によって、結果画面全体が押し潰されている状態でした。

**【解決策のステップ】**
1. **JavaScriptの表示制御をシンプルに**
   初期状態の `hidden` クラスや `<img>` タグの `display` プロパティがバッティングしていたため、クラス操作をやめて `style.display = 'block'` に統一。画像の `onload` イベントを待ってから確実に表示領域を確保するようにしました。
2. **レイアウトの抜本的見直し（fixedによる全画面化）**
   ポップアップ要素をガラポン容器（コンテナ）内部から切り離し、`<body>` 直下の全画面オーバーレイ（`fixed inset-0 z-[50]`）へと変更しました。
   これにより、どんな画面サイズでも必ず**「最前面・中央」**にポップアップが表示され、「閉じるボタンが見切れて押せない」というモバイル特有の問題も一発で解決しました。

### プログラム生成サウンドによる演出調整
当たりとハズレで雰囲気の違う音を出すために、特別な音声ファイルを用意するのではなく、Web Audio API を用いてブラウザ側で「和太鼓」や「鉦（カネ）」の音（サイン波や三角波などの合成波）を生成する処理を組み込んでいます。
これにより、当たる度に新鮮な響きが楽しめるようになりました。

## 3. 複数PCでの並行開発とGitの活用
今回の開発は、メインPCとサブPCなど、複数環境をまたいで行われた点も特徴です。
途中で「別のパソコンから予約機能（`reservation.html`）が追加され、メインブランチに統合される」といった、実際のチーム開発さながらの並行作業が発生しました。

ここで活用したのが **Git と GitHub** でした。
- 必ず作業前には `git pull origin main` で最新のコード（他で追加された機能）を取り込む。
- レイアウト修正など、機能が完成するたびに `git commit` & `git push` でクラウドに保存・連携する。
- 自分の名前やメールアドレスの「グローバル設定 (`git config --global`)」を済ませることで、どのPCからでもスムーズに安全なプッシュを可能にする。

こうした基本的な「Git運用の鉄則」を徹底することで、複数環境でも全くソースコードの競合（Conflict）を起こさずにアプリを完成させることができました。

今回は、単なる機能追加だけでなく、「画像が思い通りに縮尺を保って表示されない」というフロントエンド特有のレイアウトの悩みを、AIと相談しながら根本的な構造変更（fixedレイアウト化）で突破できたのが大きな収穫でした。

今後はこのガラポンアプリと新規追加された「予約機能」を連動させるなど、さらなる拡張が楽しみです！
