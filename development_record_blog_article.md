# AIと二人三脚！ブラウザで動く「ガラポン抽選アプリ」開発・改修記録

## はじめに
こんにちは！この記事では、ブラウザ上で動作する「ガラポン抽選アプリ」を、AIアシスタントと一緒に改修・統合した際の実績をまとめています。

機能追加からレイアウトの崩れ修正、さらにはモバイル環境での安定性向上まで、技術的なアプローチと直面した課題について振り返ります。

---

## 1. 今回の要件（やりたかったこと）
元々あったガラポンアプリをベースに、ユーザー体験（UX）の向上と実務的な予約機能の連動を目指しました。

1.  **確率と結果のシンプル化**
    - 当たり（赤色）：20%（1万円割引クーポン）
    - ハズレ（白色）：80%（残念賞）
    の2種類に絞り、期待感を醸成。
2.  **結果演出の豪華化**
    - 当たり時は `win.jpg`、ハズレ時は `lose.jpg` を明示。
    - 画面いっぱいの「紙吹雪」と、Web Audio APIによる「和太鼓＋鉦（カネ）」のサウンドで特別感を演出。
3.  **アプリ内での予約完結**
    - 当選後、そのままシームレスに来店予約ができるよう、予約フォームをメインアプリに統合。

---

## 2. 開発のハードルと解決したアプローチ

### 最大の壁：モバイルでのレイアウト崩れと画像消失
開発当初、**結果の画像が表示されず、極端に縦に潰れてしまう**という不具合が発生しました。

**【原因と解決策】**
画像を表示する親コンテナのサイズ制約や、表示・非表示を切り替えるクラス（Tailwind CSS）の干渉が原因でした。
これを解決するため、ポップアップを従来のコンテナ構造から切り離し、`<body>` 直下の**全画面オーバーレイ（fixedレイアウト）**へと変更。
これにより、モバイル端末でも「画像が正しく表示され、閉じるボタンが常に操作可能」な堅牢なUIを実現しました。

### プログラム生成サウンドによる和風演出
特別な音声ファイルを用意せず、ブラウザ側で波形を合成する Web Audio API を採用しました。
「ドドンッ」という太鼓の低音と「カーン」という高い音を組み合わせることで、和風の「お祭り感」をプログラムだけで表現しています。

## 3. 信頼性を高める「ハイブリッド送信方式」の導入
統合された予約機能において、AndroidやiPad/Safariなどのモバイル環境で「送信ボタンを押しても反応しない」という課題に直面しました。

**【技術的アプローチ】**
広告ブロックやブラウザのセキュリティ設定により、バックグラウンドでの通信（AJAX）が制限されるケースがありました。
これに対し、以下の**ハイブリッド方式**を導入して解決しました。

- **第一選択（AJAX方式）**: スムーズな画面遷移なしの送信を試行。
- **フォールバック（標準送信）**: 通信が遮断された場合、瞬時に「予備送信ボタン」を表示し、ブラウザ本来の送信機能で確実にデータを送る。

この二段構えの対策により、どんな環境のユーザーでも確実に予約を完了させることができるようになりました。

---

## 4. 複数環境での並行開発とプロジェクト整理
今回の開発は、複数のPCをまたいで並行して行われました。

- **Gitの徹底活用**: `git pull` での最新同期と、こまめな `commit` により、複数環境でも競合（Conflict）させることなくスムーズに機能を統合。
- **コードの健全化**: 開発過程で発生したコードの重複や、不要になった旧ファイル（`reservation.html`）を最終的に削除・整理。

「動けばいい」だけでなく、ファイルサイズを削減し、将来のメンテナンスにも配慮した「整理されたコード」へと昇華させることができました。
